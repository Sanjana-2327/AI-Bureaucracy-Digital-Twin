PROJECT NAME   : AI Bureaucracy Digital Twin
TYPE           : Full-Stack AI Web Application
PURPOSE        : Helps Indian citizens & entrepreneurs navigate
                 government procedures using AI-generated roadmaps.


------------------------------------------------------------
WHAT DOES THIS PROJECT DO?
------------------------------------------------------------

This is an AI-powered tool that simulates the Indian government
bureaucratic process. A user types in their goal — for example:

  "I want to open a restaurant in Delhi"
  "I want to register a startup in India"
  "I need a passport for the first time"

The AI then generates a complete, structured roadmap including:

  - Step-by-step workflow stages
  - Documents required at each stage
  - Which government authority approves each step
  - Estimated time for each stage
  - Total timeline estimation
  - Risk warnings and common pitfalls
  - What to do if things go wrong (escalation path)
  - Official acts, portals & ministries to reference
    (e.g., MCA21, Udyam, FSSAI, DigiLocker, etc.)


------------------------------------------------------------
TECH STACK
------------------------------------------------------------

FRONTEND:
  - React.js (with Vite as build tool)
  - TailwindCSS (for styling)
  - Axios (for API calls to backend)
  - Runs on: http://localhost:5173

BACKEND:
  - Python + FastAPI (REST API framework)
  - Groq SDK (to call the LLM — AI model)
  - LLM Model Used: llama-3.3-70b-versatile (via Groq Cloud)
  - python-dotenv (to load API keys from .env file)
  - Uvicorn (ASGI server to run FastAPI)
  - Runs on: http://localhost:8000

AI / LLM:
  - Provider: Groq Cloud (https://console.groq.com)
  - Model: llama-3.3-70b-versatile
  - The AI responds with structured JSON following a fixed schema

HOW IT WORKS — STEP BY STEP FLOW
------------------------------------------------------------

1. USER opens the frontend at http://localhost:5173
2. USER types a goal in the search bar (e.g. "open a bakery")
3. FRONTEND sends a POST request to:
       http://localhost:8000/generate-process
   with the body: { "user_goal": "open a bakery" }

4. BACKEND (FastAPI) receives the request in main.py
5. BACKEND sends the goal to Groq API with a detailed system
   prompt that instructs the AI to respond with structured JSON
6. AI MODEL (llama-3.3-70b-versatile) generates a complete
   governance process roadmap in JSON format
7. BACKEND validates the JSON and returns it to the frontend
8. FRONTEND displays the result in a beautiful dashboard with:
       - Process Overview card
       - Workflow stages (accordion/expandable)
       - Documents checklist
       - Risk warnings
       - Timeline estimation
       - Escalation path
       - Official references


------------------------------------------------------------
API ENDPOINTS
------------------------------------------------------------

POST  /generate-process
  - Body: { "user_goal": "your goal here" }
  - Returns: Full JSON roadmap from AI

GET   /health
  - Returns: { "status": "ok", "service": "..." }
  - Used to check if backend is alive

GET   /docs
  - Swagger UI: interactive API documentation at
    http://localhost:8000/docs


------------------------------------------------------------
HOW TO RUN THE PROJECT
------------------------------------------------------------

STEP 1 — Start the Backend:
  cd backend
  venv\Scripts\uvicorn main:app --reload --host 0.0.0.0 --port 8000

STEP 2 — Start the Frontend (new terminal):
  cd frontend
  npm run dev

STEP 3 — Open in browser:
  Frontend  → http://localhost:5173
  API Docs  → http://localhost:8000/docs

------------------------------------------------------------
KEY DESIGN DECISIONS
------------------------------------------------------------

1. STRUCTURED JSON OUTPUT
   The AI is strictly prompted to return only valid JSON.
   This makes parsing reliable and the UI consistent.

2. GROQ FOR SPEED
   Groq's inference is extremely fast (vs OpenAI),
   making the experience feel near-instant.

3. INDIAN GOVERNMENT SPECIFIC
   The AI prompt is tailored for India — it knows about
   ministries, portals (MCA21, FSSAI, Udyam, DigiLocker),
   and relevant legislation (Companies Act, GST, FSSAI Act, etc.)

4. DISCLAIMER SHOWN
   A banner reminds users this is informational only,
   not legal advice — important for compliance.

5. NO DATA STORED
   User queries are not stored anywhere. Privacy by design.
